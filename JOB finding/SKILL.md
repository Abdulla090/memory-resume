# Job Finding Skill

Use this skill when building the `JOB finding` webapp or any job-search product inside this workspace.

## Purpose
- Build a clean, professional job-finding experience.
- Keep the UI practical, fast, and mobile-safe.
- Prefer clear hierarchy, low clutter, and trustworthy copy.

## Required Design System
- Use `DESIGN.md` in this folder for visual decisions.
- Match the SubQ-inspired style when the page is technical or data-heavy.

## Working Rules
- Keep layouts responsive on mobile first.
- Use concise labels and strong scanning hierarchy.
- Avoid generic startup visuals and noisy gradients.
- Prefer card-based results, structured filters, and clear CTA states.

## Pages To Favor
- Search / discovery
- Job results
- Job details
- Saved jobs
- Application tracking
- Resume tailoring or matching

## Output Standard
- The page should feel polished, credible, and easy to act on.
- Empty states should guide the user.
- Loading states should be subtle and useful.
- On mobile, all actions must remain reachable with thumb-friendly spacing.
